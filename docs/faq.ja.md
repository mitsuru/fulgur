# FAQ

## CSS の読み込み

### Q: `--css` フラグと `<link rel="stylesheet">` のどちらを使うべき？

**HTML 内の `<link rel="stylesheet">` を推奨します。** fulgur で
スタイルシートを読み込む正規の方法です。

```html
<head>
  <link rel="stylesheet" href="./style.css">
</head>
```

### Q: なぜ？ 両者は等価ではないの？

原理的には等価のはずですが、現時点で `--css` フラグには `<link>` に
ない以下の制約があります:

- **GCPM の `@page` ルール (running ヘッダー / フッター) が適用されない
  ことがある。** `--css` で渡したスタイルシートでは margin box の内容が
  登録されず、`position: running(...)` で配置した要素が出力に現れません。

- **インライン SVG `<text>` の描画に影響することがある。** 同じ
  スタイルシートを `<link>` と `--css` の両方で読み込むと、インライン
  `<svg>` 内のテキストが描画されないケースがあります。

いずれも、fulgur の CSS パイプラインがフラグ経由のスタイルシートと
ドキュメント由来のスタイルシートを別経路で扱っていることに起因します。
`<link>` 経由なら両方の問題を完全に回避できるため、同梱の examples は
すべて `<link>` 方式を採用しています。

### Q: それでも `--css` を使う場面はある？

- **HTML を書き換えたくない (or 書き換えられない) 場合のアドホック
  なスタイル適用。** たとえば外部から受け取った HTML に印刷用 CSS を
  当てるケース。
- **HTML テンプレートが固定で、実行ごとにスタイルだけ差し替えたい
  バッチ処理。**

これらのケースでは、通常のブロック / インラインスタイリングは `--css`
でも動作します。ただし GCPM 機能やインライン SVG については、CSS
パイプラインが統一されるまで `<link>` 経由を選んでください。

### Q: 既存の `--css` ベースの example を移行するには？

1. HTML の `<head>` に
   `<link rel="stylesheet" href="./style.css">` を追加する (パスは
   HTML ファイルからの相対パス)。
2. CLI 呼び出しやビルドスクリプトから `--css style.css` 引数を削除
   する。
3. 再生成して出力を確認する。これまで欠けていた要素が表示される
   ようになった場合は、上述の制約に該当していたケースです。
